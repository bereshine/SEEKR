User Manual

Workflow: Step 1: Upload the fasta file you are interesting in analyzing using SEEKR (for more information on Fasta format for genetic sequences see https://www.ncbi.nlm.nih.gov/WebSub/html/help/fasta.html) Step 2: Choose the set of sequences to compare your file to by either selecting on of the saved options or choosing to upload another file. Step 3: Choose your normalization set Step 4: Choose your K-mer count (warning K-mers larger than 6 will not produce visualizations) Step 5: Press the submit button and the algorithms output will be visualized in the ‘Results’ tab   Visualizations: Sequence Kmer Profiles: Displays the Kmer counts for sequences on the User Set (sequences on the y axis, kmers on the x). The sequences are automatically arranged according to a k means clustering on the kmer contents of each sequence. Clicking on either the row or column labels will re-order the graph based on the values in the row or column clicked. Sequence Pearson Correlation Matrix: Display The sequences are automatically arranged according to the same k means clustering as the other visual. Clicking on either the row or column labels will re-order the graph based on the values in the row or column clicked.  

Administrator Manual

General Information: Python 3.5: Git Code Repository: https://sc.unc.edu/dept-seekr/seekr For working with the project we suggest the use of the Pycharm IDE as it simplifies much of the process, including dependency installation. This application is dependent upon several libraries for Python 3.5 (note these packages will not function with older versions of Python so be sure that your local version is up to date, we suggest using a virtual environment, instruction for setting one up can be found in Flask’s installation guide http://flask.pocoo.org/docs/0.12/installation/#virtualenv The dependencies are specified in the requirements.txt document. Windows Note: Windows does not come prepackaged with Apache and installing it can be tricky. A windows user should be able to run the program locally without Apache by only running seekrServer.py not app.py (skipping the Apache server config specified below) The application’s server environment variables are set and initialized by app.py, including --entry-point, which points the server to the main program to execute. Other configuration options are specified in skr_config.py, including constants
